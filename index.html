Quick Use

Примеры быстрого подключения плагина, сильно не вникания в настройки

Плагину для работы нужно как минимум 2 элемента: элемент с табами и элемент с контентом для каждого таба.

Элементы могут быть с любыми стилями, плагин стили не затрагивает и не навешивает (естественно кроме display:none)

Подключить плагин мы можем четырьмя способами:
  • Автоинициализация
  • Ручная инициализация
  • jQuery
  • jQuery + data-attrs
Самый простой способ подключения, просто добавить на контейнер табов класс njTabs и аттрибут data-njt-content="#quickUseContent" с селектором элемента, содержащего контент табов. Инициализация произойдет автоматически на ready ивенте и все заработает, из javascript-а ничего вызывать не нужно. 
<ul class="njTabs" data-njt-content="#exampleContent">
  <li><a href="#">MIT</a></li>
  <li><a href="#">ie7+ (необходим jQuery)</a></li>
</ul>

<div id="exampleContent">
  <div>Content of tab 1</div>
  <div>Content of tab 2</div>
<div>
При ручной инициализации код будет выглядеть следующим образом. 
<ul id="exampleTabs">
  <li><a href="#">MIT</a></li>
  <li><a href="#">ie7+ (необходим jQuery)</a></li>
</ul>

<div id="exampleContent">
  <div>Content of tab 1</div>
  <div>Content of tab 2</div>
<div>

var tabs = njTabs({tabs:'#exampleTabs','content':'exampleContent'})
или 
var tabs = new njTabs({tabs:'#exampleTabs','content':'exampleContent'})
Записи равнозначны.

Общая информация

  1. По умолчанию плагин ожидает увидеть структуру табов(ссылок) в следующем виде 
    <li><a>
    где li - элемент, на который навешивается активный класс (настройками это можно изменить). Таб для открытия по умолчанию можно указать, дав активный класс нужному элементу(как в первом примере), либо с помощью настройки start.
  2. Инициализация может быть произведена только 1 раз.
  3. Получить ссылку на инстанс плагина можно несколькими способами:
    • Использовать метод njTabs.forElement(селектор обертки табов)*; 
      njTabs.forElement('.njTabs')
    • Получить из самого элемента обертки табов 
      var tabs = $('.njTabs')[0].njTabs;
//or
var tabs = document.querySelector('.njTabs').njTabs;
    • Либо, если инициализация происходила вручную, то сразу занести в переменную 
      var tabs = njTabs({tabs:'#exampleTabs','content':'exampleContent'})
  4. Дефолтные значения настроек плагина можно изменить в объекте njTabs.defaults 
    njTabs.defaults.anim = 'myCustomAnim';
 njTabs.defaults.presentation = '.myLi';
  5. jQuery плагином можно вызывать методы 
    $('.njTabs').njTabs('destroy');
  6. Программное переключение слайдов можно организовать двумя способами:
    • Получить ссылку на инстанс и вызвать метод show с нужным индексом* 
      var tabs = document.querySelector('.njTabs').njTabs;
tabs.show(2)
    • Переключить с помощью jQuery метода 
      $('.njTabs').njTabs(2);

    * индекс считается с нуля

  7. Если при переключении слайдов указывать индекс меньше нуля, будет использован индекс - 0, если указать индекс больший, чем количество слайдов, будет использован индекс последнего слайда.
  8. Если меню (табов) многоуровневое, можно изменить настройку tabSelector, например: ".njTabs > li > a"
  9. Узнать программно активный таб можно из свойства .active инстанса табов.
  10. В инстансе также есть объекты: activeTab и prevTab, у которых есть ссылки на соответвующие дом элементы. У activeTab и prevTab есть следующие свойства:
    • tab - ссылка на таб(элемент, на который кликают)
    • index - соответсвенно индекс таба по отношению к соседним элементам
    • content - ссылка на элемент контента таба (содержимое, которые показывается)

Поддержка браузеров

Работает во всех нормальных браузерах и IE7(при подключении jQuery). В Internet Explorer младше 10 не работают анимации, т.к. нет поддержки css3 transition.

Настройки

njTabs.defaults = {
  tabs:                   '.njTabs',//(selector) default tabs wrapper selector
  content:                '',//(selector) wrapper of content divs
  
  tabSelector:            'a',//(selector) default tabs selector
  presentation:           'li',//(selector) closest element for adding active class (for ul>li>a structure)
  
  start:                  false,//(number || boolean) index (start from 0) of tab that we should show after plugin init
  active:                 'active',//(classname),!space not allowed! classname of active tab
  activeContent:          'active',//(classname),!space not allowed! classname of active tab content
  not:                    '.not-tab',//(selector) elements with this class will not trigger (for example simple link among tabs)
  
  contentSelector:        'div',//(selector) selector of tabs content (used when no target at tabs)
  
  anim:                   false,//(false || string) name of animation (see animation section)
  triggerEvent:           'click',//(event) event for change slide
  
  contentClass:           'njTabs-el',//(classname) class that will be given to every tab content element (used for animations)
  
  makeRelative:           true,//(boolean) should we make content wrapper relative? if it has static position, of course
}

data-* аттрибуты

Плагин поддерживает управление с помощью data-* аттрибутов.

Такое управление отлично подходит для настройки табов, при автоматической инициализации. Однако считываются настройки при любой инициализации, и приоритет data настроек выше приоритета настроек через js.

Следует учитывать, что плагин учитывает только настройки с префиксом data-njt-*. Для предотвращения конфликтов с настройками других плагинов.

Еще один момент, который следует учитывать: настройки, которые записаны с использованием camelCase, должны быть записаны через тире.

<ul class="njTabs" 
  data-njt-content=".njTabs-content2" 
  data-njt-anim="slideIn"
  data-njt-tab-selector="span"
  data-njt-make-relative="false"
>

Анимации

  1. Плагин не поддерживает программную анимацию, только ксс. Поэтому анимации в Internet Explorer <= 9 не работают.
  2. Тут нету жестко заданных анимаций, вы вольны создавать свои собственные(в ксс файле парочка примеров анимашек имеется: fade, scale, slideOut, slideIn).
  3. Как происходит анимация? Очень просто. Все обыгрывается на классах. Вам нужно лишь создать правильную структуру в ксс. В момент переключения таба, родительскому элементу всех элементов контента задается класс data-njt-yourAnimationName, где yourAnimationName - настройка anim. И с помощью наследования стилей, прописанных в ксс, применяется анимация. Примеры можно посмотреть в njTabs.css, там все просто.

Главное не перепутать местами порядок правил в ксс описании.
Задержки нигде задавать не нужно, плагин сам все вычислит, исходя из transition-ов.

Ивенты

Список поддерживаемых ивентов:
  • njt_hide (начало анимации скрытия)
  • njt_hidden (конец анимации скрытия)
  • njt_show (начало анимации появления)
  • njt_shown (конец анимации появления)
В колбэк ивента передается 2 аргумента: сам ивент и ссылка на инстанс. О полезностях, которые можно достать из инстанса написано тут

Ивенты нужно ловить на родителе табов (ссылок).
Пример ниже показывает как можно узнать текущий активный таб (узнавать активный таб нужно на ивентах njt_shown и njt_hidden) 

$('.njTabs').on('njt_show', function (e, instance) {
  console.log(instance.active)
})
